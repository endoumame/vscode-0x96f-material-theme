# ADR-001: Better Material Theme Darker HC のカラースキームパターンを採用

## ステータス

Accepted

## コンテキスト

0x96f Theme は独自のカラーパレットを持つ VS Code ダークテーマである。しかし、各シンタックス要素への色の割り当て（キーワード=赤系、関数=緑系、文字列=黄系など）が、広く使われている Material Theme 系の配色パターンとは大きく異なっていた。

Better Material Theme Darker High Contrast（以下 BMT）は「キーワード=紫、関数=青系、文字列=緑、タグ=赤、定数=オレンジ」という配色パターンを採用しており、多くの開発者にとって馴染みのある可読性の高い配色である。

## 決定

0x96f Theme の tokenColors における色の割り当てを、BMT のカラースキームパターンに合わせて再構成する。ただし、カラーコードの値（パレット）自体は 0x96f 独自のものを維持する。

つまり、**色の「役割分担」を BMT に揃え、色の「見た目」は 0x96f のまま**という方針を取る。

## 変更前後の色の役割の対応

| シンタックス要素 | 変更前 | 変更後 | BMT での対応色 |
|---|---|---|---|
| キーワード/ストレージ | 赤系 | 紫 | `#C792EA` (紫) |
| 演算子/その他 | シアン | シアン（変更なし） | `#89DDFF` (シアン) |
| タグ | シアン | 赤 | `#F07178` (赤) |
| 関数 | 緑 | 青（シアンブルー） | `#82AAFF` (青) |
| 定数/数値 | 紫 | オレンジ | `#F78C6C` (オレンジ) |
| 文字列 | 黄 | 緑 | `#C3E88D` (緑) |
| クラス/サポート | シアン | 黄 | `#FFCB6B` (黄) |
| 属性 | オレンジ | 紫 | `#C792EA` (紫) |

## 変更対象

- `themes/0x96f-theme.json` の `tokenColors` セクションのみ
- `colors` セクション（UI 部分）は変更しない

## 結果

- 可読性と一貫性が向上し、Material Theme 系に慣れた開発者にとって直感的な配色になった
- 0x96f 独自のカラーパレットは維持されているため、テーマとしてのアイデンティティは保たれている
