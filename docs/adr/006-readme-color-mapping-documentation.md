# ADR-006: README にカラーマッピング情報を記載

## ステータス

Accepted

## コンテキスト

テーマの配色が BMT のパターンに基づいていることや、各シンタックス要素にどの色が割り当てられているかを、テーマの利用者や将来の開発者に伝える手段がなかった。

## 決定

README.md に「Color Scheme」セクションを追加し、以下の情報を記載する：

1. BMT のカラースキームを参考にしている旨の説明
2. パレットは 0x96f 独自のものを維持している旨の説明
3. 0x96f と BMT のカラーマッピング対応表

### カラーマッピング表の内容

| シンタックス要素 | 0x96f の色 | BMT での対応色 |
|---|---|---|
| キーワード/ストレージ | `#A093E2` (紫) | `#C792EA` (紫) |
| 演算子/その他 | `#AEE8F4` (シアン) | `#89DDFF` (シアン) |
| タグ | `#FF7272` (赤) | `#F07178` (赤/ピンク) |
| 関数 | `#49CAE4` (青) | `#82AAFF` (青) |
| 定数/数値 | `#FF8787` (ライトレッド) | `#F78C6C` (オレンジ) |
| 変数パラメータ | `#FF7272` (赤) | `#FF5370` (赤) |
| 文字列 | `#BCDF59` (緑) | `#C3E88D` (緑) |
| クラス/サポート | `#FFCA58` (黄) | `#FFCB6B` (黄) |
| 属性 | `#A093E2` (紫) | `#C792EA` (紫) |
| 正規表現/エスケープ | `#AEE8F4` (シアン) | `#89DDFF` (シアン) |

## 更新ルール

- tokenColors のパレットや色の割り当てを変更した際は、README のカラーマッピング表も同時に更新する
- ADR-003 で `#FC9D6F` を `#FF8787` に変更した際の更新漏れがあったため、定数/数値の行を修正する必要がある

## 結果

- テーマの設計意図が利用者・開発者に明確に伝わるようになった
- カラーマッピング表が「真実の源」として機能し、テーマの変更時に整合性を確認できる
